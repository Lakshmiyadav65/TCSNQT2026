1. Common Errors in TCS NQT

Students often face errors due to unexpected input formats. Here's what they look like and how to spot them.

The Error
❌ ValueError (Python)
invalid literal for int() with base 10: '10 20 30'

Happens when you try to int() a whole space-separated string.

The Error
❌ ValueError (Python)
invalid literal for int() with base 10: 'six'

Happens when you try to int() a word like "six" instead of "6".

🔍 The Root Cause

The int() function in Python and Integer.parseInt() in Java only accept numeric digits. If your input contains spaces, extra characters, or words, it MUST be cleaned/processed before conversion.

🐍 Python Solutions

Single Integer Input

n = int(input().strip())

Multiple Values (Array/List)

arr = list(map(int, input().split()))

Unknown Input Format (The Ultimate Safe Method)

import sys
data = list(map(int, sys.stdin.read().split()))

Handling Word Inputs (e.g., 'six' ➜ 6)

word_to_num = {
    "zero":0, "one":1, "two":2, "three":3,
    "four":4, "five":5, "six":6,
    "seven":7, "eight":8, "nine":9
}
s = input().strip().lower()

if s.isdigit():
    print(int(s))
elif s in word_to_num:
    print(word_to_num[s])
else:
    print("Invalid")

Safe Conversion (Try-Except)

s = input().strip()
try:
    num = int(s)
except:
    print("Invalid Input")

☕ Java Solutions

Single Input (Scanner)

Scanner sc = new Scanner(System.in);
int n = sc.nextInt();

Multiple Inputs (Space Separated)

Scanner sc = new Scanner(System.in);
String[] arr = sc.nextLine().split(" ");
for(String s : arr){
    int num = Integer.parseInt(s.trim());
}

Fast I/O (BufferedReader)

BufferedReader br = new BufferedReader(
    new InputStreamReader(System.in)
);
String[] input = br.readLine().split(" ");

Handling Word Inputs in Java

Map map = new HashMap<>();
map.put("zero", 0);
map.put("one", 1);
map.put("two", 2);
map.put("six", 6);
// ... add others from mapping

String s = br.readLine().toLowerCase().trim();
if(map.containsKey(s)){
    System.out.println(map.get(s));
} else {
    System.out.println("Invalid");
}
Expert Advice

🎯 Final Exam Tips

🧱
Always Strip/Trim

Unexpected leading or trailing spaces are the #1 cause of crashes. Always use .strip() in Python or .trim() in Java.

⚡
Space Separated

Assume input might be space-separated even if the problem doesn't emphasize it. split() is your best friend for robustness.

🛡️
Safe Conversion

In problems involving mixed strings and numbers, use isdigit() (Python) or a try-catch block (Java) to avoid runtime crashes.

📑
Mapping

For word-based inputs, pre-define a dictionary or map. It's much cleaner than using 10 different if-else statements.
